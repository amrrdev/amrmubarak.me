Write a complete course on [TOPIC] for backend engineers, from zero knowledge to advanced/practical mastery.

Output as plain written prose, like an article, not a platform-formatted course.

Format rules:

No tables. If comparing two things, explain the difference in a sentence or short paragraph.
No interactive elements: no "explore how X works", "trace how X works", "let's see", "see how", diagram embeds, or clickable callouts.
No engagement questions like "do you find this helpful?" or transition lines like "pulling it together".
No boxed callouts like "Key constraint" or "Rule". If something is a key point, say it plainly in the paragraph.
Write in continuous paragraphs, like a person explaining to another engineer. Each paragraph flows into the next.
No generic AI phrasing ("it's important to note", "in today's world", "let's dive in").
Real section headers only, plain text titles, no styled cards or widgets.
Each section should build on the previous one, don't jump between topics.
Use real examples: actual function names, actual error names, actual scenarios backend engineers hit, not abstract descriptions.
Keep paragraphs short, 3-5 sentences, but connected, not sliced into disconnected one-liners.

Code examples:

Wherever a concept has a concrete equivalent in Node.js (an API call, an event, an error code, a config option), add a short Node.js code example right after explaining that concept, not batched at the end.
Each code example should be minimal and realistic, just enough to show the concept in action, with inline comments explaining what's happening at the important lines.
After each code example, add one or two sentences connecting it back to the explanation, so the code doesn't just sit there unexplained.

Content, in this exact order:

[subtopic 2]
[subtopic 3]
[subtopic 4]
...continue numbering through however many subtopics the course needs...

End each section by tying it back to something a backend engineer would actually run into in real systems, not just theory.
